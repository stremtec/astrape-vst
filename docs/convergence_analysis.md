## 1. Phase 0 (AE再構成+L1) の収束ステップ数推定

### 損失構造
```python
loss = L1(decoder(z), src) + 0.1 * MSE(z, z.detach())
```
第2項 `MSE(z, z.detach())` は常に0（zと自身のdetachの比較）→ **実質L1再構成のみ**。

### 次元圧縮率
- 入力: 88,200サンプル (2秒 × 44.1kHz)
- 潜在: 50フレーム × 768次元 = 38,400次元
- 圧縮率: 88,200 / 38,400 ≈ **2.3倍**（波形次元では2.3倍、情報量ではF³-Encoderが音韻情報のみを保持するよう設計）

### 収束に必要な実効エポック数
- VCTK: ~44時間, crop=2秒 → 約79,200サンプル/epoch
- 波形L1再構成の収束には、同種のConvNeXtV2ベースのNeural Audio Codecで**実効50-200 epoch**が典型的 (EnCodec, DAC, Mimi等の文献)
- ただし55.1Mパラメータ・batch_size=1では勾配ノイズが大きく、**実効epoch換算で1.5-3倍の係数が必要**

### 理論収束ステップ数
| 条件 | ステップ数 | 根拠 |
|:--|--:|:--|
| 最小収束 (波形が識別可能) | 80k – 150k | 約100 epoch @ batch=1 |
| 実用的収束 (L1<0.05) | 200k – 500k | 約250-630 epoch |
| 最適収束 (L1<0.03) | 500k – 1.2M | 約630-1500 epoch |

**現在のデフォルト `--steps 200000` は最小〜実用的収束の境界**。高品質な再構成には40万〜60万ステップを推奨。

### 注意点: 話者条件ゼロ埋め込み
Phase 0では `decoder(z, zeros(192))` と話者埋め込みをゼロで与える。これは「話者非依存デコーダ」を学習する設計だが、Phase 2で話者埋め込みが突然追加されるため、**デコーダはPhase 2開始時に話者条件の使い方をゼロから学習し直す必要がある**。Phase 0の収束判定はL1再構成損失のみで行うべきで、過剰適合はPhase 2の適応を遅らせる可能性がある。

## 2. Phase 1 (CFM単独): VFNが定数速度場を学習する難易度

### 問題設定
- 目標: v_θ(z_t, t, c) ≈ **v_target = z_tgt − z_src** (tに依存しない定数)
- 入力 z_t = (1-t)·z_src + t·z_tgt + σ_min·ε
- 条件 c = {speaker_emb(192), prompt_tokens(4×192), prosody(T,3)}

### 難易度評価: **中程度**（定数ターゲットのため標準CFMより容易だが、t→0での不良設定性あり）

#### 有利な点
1. **定数速度場**: tに依らず同一のv_targetを出力すればよい — 時間条件を無視することを学習すれば原理的に完全な予測が可能
2. **凍結エンコーダ**: z_src, z_tgtの分布が固定 → 分布シフトがない
3. **AdaLN-Zeroのゼロ初期化**: 各FlowBlockが初期状態で恒等写像に近く、出力ゲート `out_gate=0.01` により v≈0 から学習開始 → 安定した初期化

#### 困難な点
1. **t→0 での不良設定性**: z_0 ≈ z_src のとき、v_target = z_tgt − z_src だが z_0 は z_tgt に関する情報をほとんど含まない。VFNは**条件c（話者+韻律）のみからz_tgtを推定**しなければならない。これは条件付け経路への強い依存を生む。
2. **t→1 での冗長性**: z_1 ≈ z_tgt のとき、v_target = z_1 − z_src であり、z_1からz_srcを「差し引く」必要がある。z_srcの情報はz_1に存在しないため、再び条件cに依存。
3. **高次元出力**: 50×768=38,400次元の速度場を37.2Mパラメータで学習。パラメータ数は出力次元の約1,000倍（十分だが、batch_size=1では各次元の学習が独立に進まない）
4. **σ_min=0.001**: t=0近傍の数値安定性のための微小ノイズだが、t≈0でのz_tの多様性をほとんど生まない

### 収束推定
- CFM損失 (MSE) の典型的収束値: 初期 ~2.0 → 収束 ~0.01–0.1
- 必要ステップ: **100k – 300k** (batch_size=1換算)
- **実効エポック換算**: 約125-380 epoch

### 推奨
- Phase 1の学習率は Phase 0 と同じ `lr=2e-4` で妥当（VFNがゼロから学習のため）
- t∈[0,1]の一様サンプリングは適切だが、**t≈0とt≈1での難易度高を考慮し、β分布（α=0.5など端点重視）でのサンプリングも検討余地あり**
- σ_min=0.001はfp16で安全（fp16最小正規化数=6e-5より大）だが、t≈0での条件依存学習を促進するにはσ_min=0.01程度への増加が有益かもしれない

## 3. Phase 2 (E2E): ODEソルバ経由勾配の安定性

### ソルバ構造 (4-step Euler + Trapezoidal Corrector)
```python
# 実装上は5回のVFN forward:
z = z_src
for i in [0,1,2,3]:
    z = z + vfn(z, i/4, c) * 0.25       # Euler 4 step
z = z + vfn(z, 1.0, c) * 0.125          # Trapezoidal corrector (dt/2)
```

### 勾配経路の深さ
```
L1_loss → Decoder (6段+MRF×2, ~35層相当)
       → z_tgt_pred
       → VFN #5 (t=1.0) → VFN #4 (t=0.75) → VFN #3 (t=0.5) → VFN #2 (t=0.25) → VFN #1 (t=0.0)
       → Encoder (6段, ~12層相当)
```
- VFN1回 = 12 FlowBlock = **12層** (残差あり)
- 総深度 ≈ 5×12 + 35 + 12 ≈ **107層相当**
- **残差接続による勾配保護**: 各FlowBlockの `return x + h` により勾配は残差パスを直接逆伝播可能 → 実質的な勾配減衰は AdaLN-Zero のシグモイドゲート (初期≈0.5) とLayerScale (γ=1e-4→1.0) に依存

### 勾配品質の定量評価

#### (a) 勾配ノルム減衰
- AdaLN-Zero ゲート (sigmoid): 初期ゲート値=0 → sigmoid(0)=0.5 → 各ブロックで残差寄与が0.5倍
- 12ブロック通過後: 0.5^12 ≈ **2.4×10⁻⁴**（残差パスのみ、直接パスは1.0）
- VFN全体 (5回): 残差パスでの減衰は (0.5^12)^5 ≈ 8.3×10⁻¹⁸ → **残差パス経由の勾配は事実上消滅**
- しかし**主経路（恒等パス）**はゲートの影響を受けず、勾配は1.0で保存される
- AdaLN-Zeroのゼロ初期化は意図的に「ブロック貢献を徐々に増やす」設計 → 学習初期は恒等写像に近く、勾配は主経路を安全に流れる

#### (b) 数値精度
- fp16 (grad_scalingなし): 勾配最小値 ~6e-5 (fp16正規化数) → 上記減衰があれば勾配消失
- **fp32推奨**: VFN・Decoder・Encoderの少なくともVFNとEncoderはfp32で実行すべき
- 実際のコード: デバイス依存 (`args.device`)、MPSの場合自動的にfp32になるが明示的なdtype管理がない

#### (c) メモリ消費
| コンポーネント | 活性化メモリ (fp32, B=1) |
|:--|--:|
| Encoder (1回) | ~15 MB |
| VFN (×5回) | 5 × ~8 MB = ~40 MB |
| Decoder (1回) | ~25 MB |
| 勾配 (92M params) | ~370 MB (fp32) |
| Optimizer状態 (AdamW, 92M) | ~740 MB |
| **合計 (Phase 2)** | **~1.2 GB** |

- **batch_size=1で問題ないメモリ量** (一般的なGPUの8-24GBに十分収まる)
- batch_size=4でも ~2-3GB程度で実行可能

#### (d) ODEソルバの数値誤差

Euler法の大局誤差: O(dt) = O(1/4) = **0.25**（1次精度）
Trapezoidal補正は最終ステップのみに適用されるため、大局誤差は依然O(1/4)。

定量的評価:
- VFNが完全に定数速度場 v(z_tgt−z_src) を学習していれば、Euler法は**厳密解**を与える（定数ODEはEulerが厳密）
- 実際のVFNは時間依存の残差成分を持ち、その成分が4-step Eulerで誤差を生む
- 定数成分からのずれを δv(t) とすると、4-stepの累積誤差 ≈ δv_max × dt × 4 = δv_max
- δv_max は学習の進行とともに減少（CFM損失がvの時間依存性を罰するため）

→ **学習が進むほどODE誤差は減少する自己整合性あり**

#### (e) 勾配バイアス問題
- Eulerソルバの不正確な積分→不正確なz_tgt_pred→不正確なL1勾配→誤った方向へEncoderとVFNが更新
- これは**最適化ランドスケープを歪める**: 真の損失関数の代わりにEuler近似した損失関数を最適化
- 対策: Phase 2の後半で `n_steps` を8→16に段階的に増やす (curriculum learning)

### 推奨
1. **fp32でVFN + Encoderを実行**（勾配消失防止）
2. **Phase 2後半でODEステップ数を8に増加**（統合精度向上）
3. **RK4（実装済み）をPhase 2で使用**（2倍の計算コストで4次精度、1/16の誤差）
4. Phase 2のデフォルト `lr*0.1 = 2e-5` は適切（大規模E2Eファインチューニングの標準）

## 4. AdamW betas=(0.8, 0.9) の94Mパラメータモデルへの適合性

### 標準値との比較
| | 標準 Adam/AdamW | 本実装 |
|:--|:--|:--|
| β₁ (一次モーメント) | 0.9 | **0.8** |
| β₂ (二次モーメント) | 0.999 | **0.9** |
| 一次モーメントの時定数 | ~10 steps | **~5 steps** |
| 二次モーメントの時定数 | ~1000 steps | **~10 steps** |

### 分析

#### β₁=0.8: 低い一次モーメント
- 勾配の指数移動平均が**5ステップで約63%減衰** (標準は10ステップ)
- → モメンタム効果が弱く、最新の勾配方向により強く追従
- **利点**: Phase間の遷移（損失地形の急変）に素早く適応
- **欠点**: batch_size=1の高ノイズ勾配に対して平滑化不足 → **パラメータ更新がジッタリング**

#### β₂=0.9: 著しく低い二次モーメント
- 適応的学習率の分母 √(v_t) の推定窓が**わずか10ステップ**
- 標準のβ₂=0.999 (時定数~1000) と比較して**100倍短い**
- → 直近10ステップの勾配分散のみで学習率を調整 → **RMSprop的挙動**
- **利点**: 急激な勾配スケール変化への迅速な適応（Phase切替時に有利）
- **欠点**: 
  - バッチサイズ1では勾配分散推定が**極めて不安定** — 10ステップのサンプルでは母分散の推定誤差が √(1/10) ≈ 32% もの誤差
  - 適応的学習率が急変し、**更新の安定性が損なわれる**

### 94Mパラメータへの影響
- 大規模モデル (94M) × 低バッチサイズ (1) × 低β₂ (0.9) = **高リスク**
- 各パラメータの勾配スケールは層・位置によって3-4桁異なる（埋め込み層 vs 出力層）
- 短い適応窓では**層間の勾配スケール差を正確に推定できない**
- 特にVFN内部 (12層×512次元) では勾配の時間的変動が大きく、不正確な正規化が収束を阻害する可能性

### 定量的推奨

| フェーズ | 推奨β | 根拠 |
|:--|:--|:--|
| Phase 0 | (0.9, 0.98) | 55Mパラメータ・batch=1・安定収束が必要 |
| Phase 1 | (0.9, 0.95) | 37Mパラメータ・CFM損失は比較的安定 (MSE) |
| Phase 2 | (0.9, 0.999) | 92Mパラメータ・ODE経由勾配は特に安定性が必要 |

**現状 (0.8, 0.9) は全フェーズで攻撃的すぎる**。特にPhase 2の92Mパラメータ・ODEソルバ経由勾配では収束不安定化の主要因となりうる。

### 代替案
- 標準AdamW `betas=(0.9, 0.999)` に戻し、代わりに `lr` と `weight_decay` で調整
- またはβ₂のみ (0.9, 0.98) とし、学習率スケジューラ (cosine, warmup) を併用
- バッチサイズ1を前提とするなら、**β₂ ≥ 0.98 が強く推奨される**（勾配分散の安定推定に最低50-100ステップの窓が必要）

## 5. バッチサイズ1での勾配分散と収束速度

### 勾配分散の理論

バッチサイズBのミニバッチ勾配の共分散:
```
Cov(g_B) = (1/B) · Σ_pop  (Σ_popは母集団勾配共分散行列)
```

バッチサイズ1では:
```
Var(g_1) = Var_pop  (削減なし)
```

### 収束速度への影響

確率的最適化の収束レート (非凸, Smooth):
```
E[‖∇f(x_k)‖²] ≤ O(σ/√(η·K·B) + 1/(η·K))
```
ここでσ²は確率的勾配の分散、Kは反復回数。

- 同一Kに対し、B=1はB=32より √32 ≈ **5.7倍** 収束が遅い (分散項)
- 200kステップ @ B=1 ≈ 200k/5.7 ≈ **35k実効ステップ @ B=32**

### 実効サンプル効率

| 指標 | 値 |
|:--|--:|
| 1ステップあたり有効勾配情報 | 1サンプル (2秒音声) |
| AdamWのミニバッチ依存性 | β₁,β₂の時定数内の勾配平均 — B=1ではほぼ瞬間的 |
| 勾配クリッピング (max_norm=1.0) | 高ノイズのため**頻繁に発動**と推定 |
| クリッピング頻度推定 | 勾配ノルムの変動係数 CV≈1 と仮定 → 約30-50%のステップでクリッピング |

### 勾配クリッピングの副作用

`clip_grad_norm_(params, 1.0)` は以下を引き起こす:
1. **勾配方向の歪み**: ノルムで除算 → 大きい勾配成分が相対的に抑制
2. **実効学習率の低下**: クリッピング = 暗黙的な学習率上限 η_eff ≤ 1.0/‖g‖
3. **層間の不均衡**: 深層 (Encoder初期層) の勾配が小さく、クリッピングの影響を受けにくい → 層間の学習速度差が拡大

### 対策と推奨

1. **勾配累積 (Gradient Accumulation)**: `accumulation_steps=4~8` で実効バッチサイズを4~8に → 勾配分散を1/4~1/8に低減
   ```python
   # 修正例
   loss = loss / accumulation_steps  # 損失を正規化
   loss.backward()
   if (step + 1) % accumulation_steps == 0:
       clip_grad_norm_(params, 1.0)
       opt.step()
       opt.zero_grad()
   ```
2. **勾配クリッピング閾値の緩和**: `max_norm=5.0` または `max_norm=10.0` → クリッピング頻度を10-30%に低減
3. **学習率ウォームアップ**: 最初の1000-5000ステップで lr を線形増加 → 初期の高ノイズ勾配による発散を防止

### 収束ステップ数再推定 (B=1, 対策なし)

| フェーズ | 現状200k steps | 対策後 (grad_acc=4) |
|:--|:--|:--|
| Phase 0 | 最小収束 (~150k) | 良好収束 (~75-100k相当) |
| Phase 1 | 部分的収束 | 良好収束 (~50-75k相当) |
| Phase 2 | 収束不足の可能性高 | 安定収束 (~80-120k相当) |

**200kステップで全Phaseを回す場合、勾配累積なしではPhase 2の収束が不十分になる可能性が高い**。

## 6. 3-Phase間の損失スケール差と勾配更新への影響

### 各Phaseの典型的損失スケール

| Phase | 損失関数 | 初期値 | 収束値 | 勾配ノルム (概算) |
|:--|:--|:--|:--|:--|
| Phase 0 | L1(waveform) | 0.3–0.5 | 0.02–0.08 | 0.5–2.0 |
| Phase 1 | MSE(v_pred, v_target) | 1.0–3.0 | 0.01–0.1 | 0.2–1.0 |
| Phase 2 | L1(decoder(z_cfm), tgt) | 0.3–0.8 | 0.03–0.12 | 0.1–0.5 |

### 損失スケールの互換性

- 全Phaseで損失は**0.01–3.0の範囲**に収まり、同一の `lr=2e-4` で概ね互換
- Phase 1のCFM損失 (MSE) はPhase 0/2のL1損失より1桁大きくなりうるが、VFNのみの最適化のため独立
- **重大な不整合**: Phase 2では `lr*0.1 = 2e-5` を使用 (Phase 0の1/10)

### Phase 2の勾配スケール問題

Phase 2の勾配経路:
```
∂L/∂θ_enc = ∂L/∂z_tgt_pred · ∂z_tgt_pred/∂z_src · ∂z_src/∂θ_enc
           = ∂L1/∂out · ∂decoder/∂z · (∂ODE_solver/∂z_src) · ∂encoder/∂θ_enc
```

各項のスケール:
1. `∂L1/∂out`: L1勾配 → 各サンプルで ±1 (波形値あたり) → **O(1)**
2. `∂decoder/∂z`: Decoderのヤコビアン (28.7Mパラメータ経由) → FiLM条件付け・TransposedConvを通じて減衰 → **O(0.01–0.1)**
3. `∂ODE_solver/∂z_src`: VFNを5回通過するODEの感度
   - Euler: `∂z_tgt/∂z_src = I + Σ ∂vfn/∂z · dt`
   - VFNのヤコビアン: AdaLN-Zero × 12ブロック, out_gate=0.01 → **O(0.01–0.1)**
4. `∂encoder/∂θ_enc`: Encoderの勾配 → 標準的なスケール **O(0.1–1.0)**

**積**: O(1) × O(0.01) × O(0.01) × O(0.1) = **O(10⁻⁵)** → **Encoder勾配が極小**

### 層別勾配スケール推定

| コンポーネント | 勾配相対スケール | 問題 |
|:--|:--|:--|
| Decoder 最終層 | 1.0 (基準) | — |
| Decoder 中間層 | 0.1–0.5 | 標準的な減衰 |
| VFN 出力層 | 0.05–0.2 | out_gate=0.01 → 減衰 |
| VFN 中間層 | 0.001–0.01 | AdaLN-Zeroゲート + 5回通過 |
| Encoder 最終層 | 0.0001–0.001 | **深刻な勾配消失** |
| Encoder 初期層 | 0.00001–0.0001 | **事実上学習不能** |

### 解決策

1. **層別学習率 (Layer-wise LR)**:
   - Encoder: lr = 5e-5 (Phase 2基準の2.5倍)
   - VFN: lr = 2e-5 (基準)
   - Decoder: lr = 1e-5 (Encoderより遅く、VFNの出力に適応させる)

2. **VFN out_gate の初期値増加**:
   - 現在 `out_gate = 0.01` → `0.1` または `1.0` に変更
   - Phase 1では0.01で問題ない（直接CFM損失）が、Phase 2では勾配の通り道として機能するため増加が必要
   - **推奨**: Phase 2開始時に `out_gate` を0.1に再初期化

3. **Phase 2でのVFN凍結期間 (Warmup)**:
   - 最初の10k-20kステップはVFNを凍結し、Encoder+Decoderのみ更新
   - DecoderがVFN出力に適応した後、VFNを解凍
   - これによりEncoder勾配が安定化

4. **Decoder内の話者埋め込み経路の勾配スケール**:
   - Phase 0では話者埋め込み=0 → FiLMが非活性
   - Phase 2では話者埋め込み≠0 → Decoderの活性化分布が変化
   - Phase 2開始時に**Decoder出力が大きく乱れる可能性あり** → Phase 2最初の数千ステップで損失スパイクが予想される

### 損失スケールのPhase間遷移

```
Phase 0 (L1≈0.05) ──→ Phase 1 (MSE≈1.5) ──→ Phase 2 (L1≈0.3→0.05)
        ↑                    ↑                      ↑
    Enc+Dec更新          VFNのみ更新           Enc+Dec+VFN更新
    勾配: 安定            勾配: 安定            勾配: Encoder消失の危険
```

**Phase 0→1の遷移**: 問題なし（最適化対象が完全に切り替わる）
**Phase 1→2の遷移**: **高リスク** — Encoderが凍結→突然学習再開 + 勾配スケールが100-1000倍小さい

## 総合的推奨事項

### 優先度: 高 🔴
1. **β₂を0.9→0.98以上に変更**: batch_size=1での勾配分散推定の安定化に必須
2. **VFN out_gateを0.01→0.1に**: Phase 2のEncoder勾配消失の主要因を緩和
3. **勾配累積の導入**: `accumulation_steps=4` で実効バッチサイズ4を確保

### 優先度: 中 🟡
4. **ODEステップ数のカリキュラム**: Phase 2前半=4, 後半=8
5. **Phase 2 warmup**: 最初の10kステップはVFN凍結
6. **層別学習率**: Encoderに2-3倍のlr
7. **Phase 0のステップ数**: 最低400k（現状200kは不足気味）

### 優先度: 低 🟢
8. **RK4ソルバの採用**: 実装済み `solve_cfm_rk4` をPhase 2後半で使用
9. **t分布の調整**: Phase 1でβ分布サンプリングを検討
10. **勾配クリッピング閾値**: 1.0→5.0 に緩和
